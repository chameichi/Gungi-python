# 軍議 with Python

『HUNTER × HUNTER』に登場するボードゲーム「軍議 (Gungi)」を Python で実装する個人プロジェクト。
ルール検証エンジン・PySide6 製 GUI・将来の AI エンジン用プロトコル (UGI) を含みます。

## 機能

- 9×9 盤面、14 種駒、最大 3 段ツケに対応した軍議のルールロジック
- 段位差ルール（ツケていない駒はツケ駒を取れない）の実装
- 布陣段階 / 対局段階の 2 フェーズ管理（先手→後手の済み宣言、両陣の帥アタリ検知）
- PySide6 製 GUI: 木目盤 + 白黒の丸駒、棋譜パネル、戻す/進める、編集モード
- 編集モード: 盤・手駒・捕獲駒・手番・段階を自由に編集（ゲーム規則の制約なし）
- システムテーマ追従（ライト/ダーク両対応）
- UGI（Universal Gungi Interface）プロトコル定義: USI 準拠の文字列形式で局面・手をシリアライズ

## 起動方法

[uv](https://github.com/astral-sh/uv) を利用しています。

```bash
# 依存関係のインストール
uv sync

# GUI 起動
uv run python gui.py

# CLI デモ (盤面と数手を表示)
uv run python main.py
```

`Ctrl+C` で GUI を正常終了できます。

## プロジェクト構成

| ファイル | 役割 |
|---|---|
| `pieces.py` | 駒種・色・移動パターン・難易度設定 |
| `board.py` | 9×9 盤面・スタック管理・移動可能マス計算 |
| `game.py` | 対局状態 (Game)・ルール検証・布陣段階・スナップショット履歴 |
| `protocol.py` | UGI プロトコル (GFEN・アクション記法・ハンドラ基底) |
| `gui.py` | PySide6 製 GUI (`BoardWidget`・`HandPanel`・`MoveListPanel` 他) |
| `main.py` | CLI デモエントリポイント |

層構造:

```
GUI (gui.py)
   ↓
コアロジック (game.py / board.py / pieces.py)
   ↓
プロトコル変換 (protocol.py) ↔ AI エンジン (将来 engine_*.py)
```

## 難易度モード

ルールブック page 14 準拠で 4 段階を実装。

| 難易度 | 初期配置 | 特殊駒 | ツケ最大 | 帥ツケ |
|---|---|---|---|---|
| 入門編 | 初期配置① (固定) | なし | 2 段 | 不可 |
| 初級編 | 初期配置② (固定) | 弓のみ | 2 段 | 不可 |
| 中級編 | 自由配置 (布陣段階) | 全種 | 2 段 | 可 |
| 上級編 | 自由配置 (布陣段階) | 全種 | 3 段 | 可 |

布陣段階のルール:

- 先後交互に「帥」から 1 駒ずつ自陣 3 行目までに配置
- 配置の度に手番交代（相手が「済み」宣言済みなら同じ側が継続）
- 先手が「済み」を宣言してから後手が「済み」を宣言した時点で対局開始
- 両陣の帥がアタリでないことを確認してから対局開始

## 操作

### 対局モード

- 自駒のマスをクリック → 移動候補マスがハイライト
- 行き先マスをクリック → 取る/ツケる の選択ポップアップ（ambiguous な場合）
- 手駒/捕獲駒をクリック → 新（あらた）モード → 盤面マスをクリックで打つ
- ツールバー: `◀◀ 開始 / ◀ 戻す / 進める ▶ / 最新 ▶▶` で棋譜ナビ
- `←` `→` キーでも 戻す/進める可能、`Esc` で選択解除

### 編集モード（ゲーム独立）

- マスクリック → `EditDialog` で 配置/削除
- 手駒・捕獲駒パネルの「＋」 → `SimpleAddDialog` で駒を追加
- 手駒・捕獲駒の駒をクリック → 削除
- ツールバー: `盤面クリア / 手番切替 / 段階切替`（PLACEMENT⇔PLAY 強制）

### 新規対局

ツールバー「新規対局 ▾」プルダウンから `入門 / 初級 / 中級 / 上級` を選択。
中級・上級は空盤の布陣段階から開始するので、駒を交互に配置して「配置完了」で対局を開始する。

## UGI プロトコル

USI に倣った Gungi 用プロトコル。詳細は `protocol.py` 冒頭の docstring 参照。

### 駒コード

| 駒 | コード | 駒 | コード |
|---|---|---|---|
| 帥 | `Su` | 兵 | `Hy` |
| 大 | `Ta` | 弓 | `Ym` |
| 中 | `Cj` | 筒 | `Tt` |
| 小 | `Sh` | 砲 | `Od` |
| 侍 | `Sm` | 謀 | `Bo` |
| 槍 | `Ya` | 馬 | `Ki` |
| 忍 | `Sn` | 砦 | `To` |

色プレフィクス: `w` (先手) / `b` (後手)

### アクション記法 (USI 準拠)

| 種類 | 記法 | 例 |
|---|---|---|
| 移動 | `<sx><sy><dx><dy>` | `4243` |
| ツケ | `<sx><sy><dx><dy>+` | `4243+` (`+` を「成り」から「ツケ」choice に流用) |
| 打ち | `<駒>*<dx><dy>` | `Hy*43` (PLACEMENT/PLAY phase で布陣/新を判別) |
| 配置完了 | `done` | |
| 投了 | `resign` | |
| 待機 | `null` | |

### GFEN (Gungi Forsyth-Edwards Notation)

```
<board> <turn> <phase> <hand_w> <hand_b> <cap_w> <cap_b> <done_w> <done_b> <ply>
```

`board` は 9 段を `/` 区切り、各段を `|` 区切り、空きマスは数字で圧縮。
`startpos:intro` 等のショートカットで難易度別の開始局面を指定可能。

### コマンド (USI 準拠)

GUI → エンジン: `ugi`, `isready`, `uginewgame`, `position`, `go`, `stop`, `setoption`, `quit`
エンジン → GUI: `id`, `ugiok`, `readyok`, `option`, `info`, `bestmove`
