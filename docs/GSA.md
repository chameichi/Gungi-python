# GSA - Gungi Standard Action

CSA 流の手表記を軍儀向けに本格設計した棋譜フォーマット。
1 行のヘッダー群 + 1 行 1 手の本文という構成で、機械可読かつ
CSA 系ツールの応用が利く設計。

## 概要

| 項目 | 仕様 |
|---|---|
| ファイル拡張子 | `.gsa` |
| エンコーディング | UTF-8 |
| 改行コード | LF (`\n`) |
| 行ベース | 1 行 = 1 メタ情報 or 1 手 |
| 設計の源流 | 将棋 CSA フォーマットを軍儀向けに再設計 |

## ファイル構造

```
V Gungi-1            ← バージョン (必須)
$KEY:value           ← メタ情報 ($-prefix, 任意・複数)
%MOVES               ← 手順セクション開始 (必須)
<手>                  ← 1 行 1 手
<手>
...
%END                 ← ファイル終了マーカ (必須)
```

セクション境界:
- ヘッダーセクション: ファイル冒頭〜`%MOVES` 行
- 手順セクション: `%MOVES` の次行〜`%END` 行
- `'` で始まる行はコメント (パーサは無視)

## ヘッダー

### バージョン (`V`)

```
V Gungi-1
```

将来形式が変わったときの互換性判定用。

### メタ情報 (`$<KEY>:<value>`)

| キー | 必須 | 意味 |
|---|---|---|
| `$EVENT` | 任意 | イベント名 (例: `対局`) |
| `$DATE` | 任意 | 日時 (`YYYY/MM/DD HH:MM:SS`) |
| `$DIFFICULTY` | 任意 | 難易度 (`INTRODUCTORY` / `BEGINNER` / `INTERMEDIATE` / `ADVANCED`) |
| `$INITIAL_GFEN` | **必須** | 開始局面の GFEN (詳細は [GFEN](GFEN.md) 参照) |
| `$RESULT` | 任意 | 結果 (`*` / `1-0` / `0-1`) |

`$INITIAL_GFEN` が無いと load 失敗。

## 手の記法

すべて **区切りなしの密記法** (CSA 流)。

### 共通フォーマット

```
<side><from><to><piece>[stack_suffix]
```

- `<side>`: `+` = 先手 (白) / `-` = 後手 (黒)
- `<from>`: 2 桁の 1-based 座標 (移動元)
- `<to>`: 2 桁の 1-based 座標 (移動先)
- `<piece>`: 2 文字 (大文字) の駒コード
- `<stack_suffix>`: 末尾 `^` でスタック (ツケ)、無ければ移動

### 手の種類

| 種類 | 記法 | 例 | 説明 |
|---|---|---|---|
| 移動 | `±AABBPP` | `+3142SH` | 先手が (3,1) から (4,2) へ Sh を移動 |
| ツケ | `±AABBPP^` | `+3142SH^` | 先手が (3,1) の Sh を (4,2) の上にスタック |
| 打ち | `±00BBPP` | `+0042SU` | 先手が手駒の Su を (4,2) に打つ (CSA 流の from=00) |

### 打ちの phase

打ち (`±00BBPP`) は load 時の局面の phase で意味が決まる:
- **PLACEMENT** → 布陣として手駒を盤面へ
- **PLAY** → 「新 (あらた)」として手駒/捕獲駒を打つ

## 駒コード

PIECE_CODE の大文字版 2 字。

| 駒 | コード | 駒 | コード |
|---|---|---|---|
| 帥 | `SU` | 兵 | `HY` |
| 大 | `TA` | 弓 | `YM` |
| 中 | `CJ` | 筒 | `TT` |
| 小 | `SH` | 砲 | `OD` |
| 侍 | `SM` | 謀 | `BO` |
| 槍 | `YA` | 馬 | `KI` |
| 忍 | `SN` | 砦 | `TO` |

## 座標

- **1-based** 2 桁 (`<col><row>`、各 1〜9)
- 内部 (0-8) → GSA: 各成分 +1
- 「from = `00`」は手駒から (打ち) を示す予約値 (実際の盤上には存在しない)

> GFEN や UGI のアクション記法は 0-based。**GSA だけが 1-based** なので変換時に注意。

例:
- 内部 `(3, 1)` → GSA `42`
- 内部 `(0, 0)` → GSA `11`
- 内部 `(8, 8)` → GSA `99`

## 特殊コマンド

手番側のアクションのうち、座標を伴わないもの。

| コマンド | 意味 | 例 |
|---|---|---|
| `±%DONE` | 当該手番が布陣段階を完了 | `+%DONE` (先手が完了) |
| `±%TORYO` | 当該手番が投了 | `-%TORYO` (後手が投了) |

`+` / `-` の手番プレフィクスは特殊コマンドにも必須。

## 完全な例

```
V Gungi-1
$EVENT:対局
$DATE:2026/05/11 20:42:31
$DIFFICULTY:INTRODUCTORY
$INITIAL_GFEN:3|bCj|bSu|bTa|3/1|bSn|2|bYa|2|bSn|1/bHy|1|bTo|bSm|bHy|bSm|bTo|1|bHy/9/9/9/wHy|1|wTo|wSm|wHy|wSm|wTo|1|wHy/1|wSn|2|wYa|2|wSn|1/3|wTa|wSu|wCj|3 w pa Sh,Sh,Ya,Ya,Ki,Ki,Hy Sh,Sh,Ya,Ya,Ki,Ki,Hy - - 0 0 0 intro
$RESULT:*
%MOVES
+4131TA              ← 先手が (4,1) から (3,1) へ大を移動
-6979TA              ← 後手が (6,9) から (7,9) へ大を移動
+5141SU              ← 先手が (5,1) から (4,1) へ帥を移動
-5969SU              ← 後手が (5,9) から (6,9) へ帥を移動
+6172CJ              ← 先手が (6,1) から (7,2) へ中を移動
-4938CJ              ← 後手が (4,9) から (3,8) へ中を移動
%END
```

(座標例は説明のための仮想列。実際の合法手とは限らない)

## 本家 CSA との違い

CSA は将棋専用なので、軍儀固有の概念は流用できず独自設計している:

| 観点 | CSA (将棋) | GSA (軍儀) |
|---|---|---|
| 盤面 | 9×9 平面 | 9×9 + 最大 3 段スタック |
| 駒種 | 8 種 (歩香桂銀金角飛王) | 14 種 |
| 移動の記法 | `+7776FU` (先手 7→7 マス目 + 駒種) | `+3142SH` (同様、座標 2 桁ずつ) |
| 「成り」 | 移動先で `+` 表記 | 該当概念なし。代わりに「ツケ」を末尾 `^` で表現 |
| 打ち (持駒) | `+0076FU` (from=00) | 同じ (`+0042SU`) |
| 詰み・投了 | `%TORYO` 等 | `±%TORYO` (手番付き) |
| 布陣段階 | 該当なし | `±%DONE` で完了宣言 |
| 座標基準 | 1-based (筋・段) | 1-based (col・row) |

## 他フォーマットとの比較

| フォーマット | 拡張子 | 用途 |
|---|---|---|
| **GSA** | `.gsa` | 機械可読、CSA 系ツールに馴染む密記法 |
| **Gungi 棋譜 (PSN/KIF 風)** | `.gungi` | 人間が読みやすい番号付きリスト形式 |
| **JSON** | `.json` | プログラム連携・diff しやすい |

GUI の「棋譜 ▾」メニューから任意のフォーマットで保存/読込できる。
